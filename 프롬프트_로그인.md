### 로그인 페이지 생성 프롬프트
메인 페이지 접근시 로그인 후 접근할 수 있도록 로그인 페이지를 별도로 만들어줘.

1. 로그인 필수 조건
- 로그인하지 않은 사용자는 어떤 내부 페이지도 볼 수 없어야 함

2. 로그인 기능
브라우저를 종료하거나 PC를 재부팅해도, 사용자가 직접 로그아웃하지 않는 한 로그인 상태가 유지되도록 구현한다.

구현 조건:
- 로그인 성공 시 사용자 정보를 session에 저장한다.
- session ID는 브라우저 쿠키에 저장한다.
- 쿠키는 브라우저 종료 후에도 유지되는 persistent cookie로 설정한다.
- session 저장소는 서버 메모리가 아니라 DB, Redis, 파일 등 재시작 후에도 유지 가능한 저장소를 사용한다.
- 로그아웃 시 session과 쿠키를 모두 삭제한다.
- 보안을 위해 session 만료 기간을 설정하되, 기본값은 30일로 한다.
- 관리자(admin)와 일반 사용자(user) 권한은 구분하지 않는다.

3. 환경 변수 관리
- 모든 계정 정보 및 환경 설정은 `.env` 파일에서 관리
- 비밀번호, SECRET_KEY 등을 코드에 직접 하드코딩하지 않음

.env 예시 포함:
- ADMIN_ID
- ADMIN_PASSWORD
- USER_ID
- USER_PASSWORD
- SECRET_KEY

4. 보안 강화
- Flask session 기반 인증 사용
- `.env` 기반 설정 로딩
- `.gitignore`에 `.env` 포함
- 비밀번호를 코드에 직접 작성하지 않음
- 모든 내부 라우트에 로그인 검증 적용
- admin 전용 기능은 서버 라우트에서도 반드시 권한 검증
- 단순히 버튼을 숨기는 방식만으로 보안 처리하지 말 것
